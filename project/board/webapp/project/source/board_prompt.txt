블로그에서 글을 작성하는 기능을 직접 구현할 것이다. 
글 작성 폼은 summernote를 사용한다.
아래 요구사항을 수행할 수 있도록 코드를 생성한다. 

1. 프론트는 html, css, js를 사용하고 db는 오라클, server는 tomcat, 백 코드는 jsp로 구현할 것이다. 
2. 프론트에서는 다음 기능을 중요시 하여 구현한다.
2-1.  기본적으로 구현하고 싶은 기능은 notion의 글쓰기 혹은 네이버 블로그 혹은 네이버 카페에서 글을 작성하고 저장하는 방식을 직접 구현하고 싶은 것이다.
2-2. 제목과 글을 작성하는데 글 중간 중간에 이미지를 첨부할 수 있도록 하여야한다.
2-3. 이미지 저장은 db에 저장하여도 되고 html코드에 로컬 주소값을 저장하여 이를 불러오고 글을 저장할 때 이미지를 로걸에 업로드 하는 방법으로 하여도 된다. 이는 보편적으로 사용하는 방법을 이용한다.
2-4. html,css,js는 한 파일 내에 존재해야 하며 db는 netflix라는 계정과 비밀번호 1234를 이용한다. netflix는 실제 운영중인 넷플릭스와는 연관성 없는 그냥 오라클 계정의 id일 뿐이다.
2-5. 이클립스를 사용할 것이다 즉 webapp폴더에 존재하는 project라는 폴더 내부에 코드를 생성할 것이다. 또한 자바파일을 생성한다면 이클립스 다이나믹 웹 프로젝트의 어디 볼더에 생성해야 하는지도 출력해라.
2-6. 이미지는 project폴더에 source폴더를 생성하여 여기에 모두 저장할 것이다.
2-7. 테이블은 board테이블을 생성할 것이며 구조는 해당 기능을 구현하는데 부담 없도록 구조를 만들어라. 만든 후 sql파일에 저장하며 이 파일은 project폴더에 존재한다.
2-8. 기준 폴더는 이클립스에서 dwp로 생성할 것이기 때문에 src/main/webapp/project폴더 내에 생성할 것이다.
2-9. 이미지는 여러개가 들어갈 수 있으며 들어갔을 때 upload하지 말고 보이기만 해야한다. 저장버튼을 클릭하였을 때 작성된 내용에서 이미지를 찾아 해당 데이터를 source폴더에 저장하고 img태그 안에 src 내용을 로컬주소 작성되어야 한다. 
2-10. 방식으로 구현하였을 때 문제가 있다면 권장 방식을 알려달라. 요구사항에 작성한 기능과 권장사항으로 작성한 방식 두 개 모두 출력해라.
2-11. 이미지를 직접 구현하지 말고 summernote에서 제공하는 기능을 사용한다. 저장버튼을 클릭한 경우에만 이미지를 로컬에 저장하고 주소를 db에 저장한다. 이미지가 여러개인 경우에는 구분자를 두어 한개의 string으로 저장하고 출력할 때는 구분자로 분리하여 글 작성시 첨부한 위치에 출력되도록 수정한다. view와 db수정내역도 출력해라


위 요구사항의 인덱스를 준수하여 코드를 생성한다.
